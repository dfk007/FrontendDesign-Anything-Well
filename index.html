Section 01 · Markdown Skill Files
The five design systems
Each .md file is a complete, self-contained design specification. Load it into your agent and it will reproduce that aesthetic faithfully.
01 DeepNavy-HighQuality-Terminal-Aesthetics.md · the system applied to this page ▼

Deep navy-slate editorial dark UI with electric teal accent, dual-axis variable-font typography (Fraunces serif + Geist sans + JetBrains Mono), and a scanline/ambient-glow atmosphere. Three-column masthead/footer, asymmetric 1.3fr/1fr hero grid, ruled meta strip, accordion Q&A panels with left-border tier accents. Full data-theme light/dark toggle.

Use for: interview prep guides, AI platform dashboards, internal knowledge bases, document-dense SPAs. The current page is built on this system.

✓ This page = live example
★ Default skill for new projects
02 skill.md · the original editorial craft system ▼

Warm parchment background with a 40px blueprint grid and SVG noise-grain overlay. Newspaper-style masthead with double-rule top border, Fraunces display headlines, lifted-paper cards with ::before shadow-offset pseudo-element, forest-green answer blocks with mustard left border, dark-brown code blocks. Zero border-radius on cards. Light/dark toggle (◐) with 0.4s background transition.

Use for: technical journals, field manuals, content that should feel like a 1970s print publication. The default system for open-ended editorial work.

✓ Most widely applied
03 DeepNavy-Fraunces-Pro.md · editorial-technical, the ancestor of #01 ▼

Production-grade design system combining editorial typographic refinement with technical/terminal culture. Deep navy backgrounds, electric teal accent, Fraunces + Geist + JetBrains Mono stack, grid-line texture, scanline overlay, collapsible accordion panels, light/dark theme toggle.

Use for: dashboards, knowledge bases, interview prep tools, documentation sites, reference guides, data-heavy reports, developer portals, internal tooling UIs. Authoritative, not flashy.

✓ Predecessor of v2 system
04 Dark-Gold-Editorial-Report.md · financial-journalism meets technical docs ▼

Dark-masthead editorial report system. Primary accent: warm gold #c8922a on an ink-dark navy masthead (#0d1117), over a warm off-white parchment body (#f7f6f2). Font stack: DM Serif Display + DM Mono + Space Grotesk. Signature patterns: gold eyebrow labels in DM Mono ALL-CAPS, full-bleed dark-ink inversion panels, sticky monospace navigation tabs with gold underline hover, 3px top-border color-coded role cards, vertical timeline with gold/teal phase dots, two-tone colophon footer.

Use for: editorial reports, career guides, technical roadmaps, research documents, dashboards. The system that projects authority and analytical depth.

✓ Has a live HTML example
★ Best for long-form reports
05 CyberPunk-Animated-Terminal-v1.md & v2 · hacker terminal / network ops ▼

Two variants of the cyberpunk terminal aesthetic. Deep void-black backgrounds with animated green grid scrolling vertically, neon green / cyan / magenta triad, glowing box-shadow and text-shadow as the primary depth mechanism. v1 uses a single shared .md (which is also the rendered HTML). v2 uses Orbitron display + Fira Code monospace, slightly rounded corners (6–8px), animated scanning chromatic gradient stripe on every card top, sticky header with glow. Permanently dark — light theme is incompatible.

Use for: security dashboards, network operations centers, dev-tool landing pages, sci-fi / futuristic interfaces. Everything is alive with data.

✓ Both have live HTML examples
★ Dark-only · no theme toggle

Section 03 · Workflow
How to use these skills
No CLI, no build step, no configuration. Each skill is a single markdown file that any sufficiently capable LLM can read and apply.
11 Step 1 — load the skill into your agent ▼

Most agents that support a skill tool will pick up frontend-design/<name>.md automatically. If yours doesn't, paste the entire file into the system prompt or context.

For Claude Code specifically, use the Claude-Skills/ variants (e.g. Claude-default-SKILL.md or any .skill bundle).

✓ Auto-discovery usually works
12 Step 2 — provide the HTML you want revamped ▼

Either as a file path the agent can read, or pasted inline. The skill is opinionated but not destructive — semantic content, copy, and code blocks are preserved. Only the design system changes.

✓ Content is preserved
13 Step 3 — write a short prompt ▼

The prompt is usually just:

Using the frontend-design skill at frontend-design/<name>.md, update this HTML file: <path>. Preserve all content, copy, and code blocks. Only the design system (CSS + structure where needed) should change.

If your agent doesn't auto-discover skill files, paste the entire skill body into the prompt and reference it.

✓ Minimal prompt = best results
★ Be explicit about preservation
14 Step 4 — review the output & extend if needed ▼

Each skill is one file. Edit it freely:

  • New colour tokens — add to :root and [data-theme="dark"] in the colour section.
  • New components — add a section and renumber.
  • Different aesthetic — the load-bearing rules (zero border-radius, no blurred shadows, no glassmorphism, three-font stack) are what make the system look the way it looks. Override them and you get a different design system.

If you fork a skill, keep its most recognisable fingerprint (e.g. the em rust-italic rule for the editorial craft system, or the teal top-border for the deep-navy system) — dropping it makes the output generic.

✓ Edit freely · MIT licensed
Section 04 · Notes on This Index
Design notes & conventions
This page is itself a worked example of the DeepNavy-HighQuality-Terminal-Aesthetics system applied to a content-inventory / index use case.
15 How this index was composed ▼

The page is a single index.html with no build step, no framework, no external assets except three Google Fonts: Fraunces, Geist, JetBrains Mono.

It uses every load-bearing component from the DeepNavy spec:

  • Three-column masthead with teal top border
  • Asymmetric 1.3fr/1fr hero with // co-name eyebrow
  • Ruled meta strip with 6 stat cells
  • Search bar with teal left-border accent
  • Tabbed nav with tier-colored active underlines
  • 4 tiered accordion sections (easy · intermediate · hard · other)
  • Single-open accordion behavior with rotating arrow
  • Cross-section full-text search filter
  • Three-column footer with teal top border
  • Colophon crediting the type system
  • Theme toggle (◑) with localStorage persistence
  • Scanline + ambient-glow atmospheric layers
  • Staggered page-load entrance animation
  • Print stylesheet that expands all accordions and hides navigation
  • Responsive collapse at 860px
✓ Reference implementation
★ Use as scaffolding template
