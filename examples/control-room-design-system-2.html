Phases The full plan, in order
01
Persistence & library
Close the tab, keep the playlist. Resume from where you left off.
Biggest QoL
+

The single biggest UX gap: closing the tab loses the entire playlist. Fix that first.

  • IndexedDB media library — store loaded files (or references + blob) so the playlist survives reloads. Use a small wrapper or the idb library via CDN.
  • Auto-save current state — debounced save of: currentIndex, currentTime, playbackRate, volume, muted, isShuffle.
  • Resume on load — on startup, restore the library, then seek to the saved time of the last-played file.
  • Clear library button + confirmation.
02
Playback polish
Loop, volume keys, persistence. Round out the keyboard surface.
Shortcuts
+
  • Loop one / loop all — new L shortcut, button next to Shuffle.
  • Volume control — ↑/↓ keys (5% steps), visual volume bar.
  • Save volume/speed to localStorage — they're global preferences, not per-playlist.
  • Time display in shortcuts UI — surface current currentTime / duration somewhere subtle (footer of shortcut help).
  • Click-to-seek on the video — optional, depends on whether you want native video controls hidden.
03
Playlist & search
Sort, drag-reorder, auto-scroll. Make the queue feel like a queue.
Queue UX
+
  • Persistent sort — sort by name / date added / duration. Saved in localStorage.
  • Active-item auto-scroll — if you hit "Next" while the active row is off-screen, scroll it into view.
  • Drag-to-reorder — HTML5 DnD on playlist items.
  • Remove from playlist — small ✕ on hover, with optional "remove and revoke object URL" cleanup.
  • Group by type — collapse video / audio sections, optional toggle.
04
Subtitles & captions
Sidecar .srt / .vtt files. Toggle with C.
Captions
+
  • Load .srt / .vtt file — paired to current video, or sidecar file (same basename).
  • Toggle subtitles — C shortcut.
  • Style controls — font size, color, background, position. Persist to localStorage.
  • Auto-detect sidecar — when a video is added, look for <name>.vtt in the same File batch.
05
Quality, speed & chapters
Speed presets, chapter overlay, sleep timer.
Advanced
+
  • Playback speed presets — 0.5x / 1x / 1.25x / 1.5x / 2x menu, plus the [/] fine adjust.
  • Speed persistence — per-file or global (your call; default global).
  • Chapter support — parse YouTube-style chapter comments or sidecar .chapters file, show as a timeline overlay.
  • Sleep timer — pause after N minutes. Useful for long playlists.
06
Visuals & information
Richer properties popup, theme toggle, custom background.
Polish
+
  • Media-properties popup upgrade — add: codec, bitrate (from MediaSource/getVideoPlaybackQuality where available), file size, last played timestamp.
  • Mini waveform / timeline preview for audio files using a Web Worker + OfflineAudioContext. Skip if you don't listen to audio here.
  • Theme toggle — dark (current) / light / system. Single CSS variable swap.
  • Custom background — user-supplied image for the page (saved as object URL in IndexedDB).
07
Power features
Picture-in-picture, theater mode, cast, cheat sheet.
Power user
+
  • Picture-in-Picture — I shortcut, uses video.requestPictureInPicture().
  • Theater mode — hides playlist, expands player. T shortcut.
  • Cast support — Presentation API (presentation.requestShow()) for Chrome → TV. Best-effort.
  • Keyboard cheat sheet — ? toggles an in-app overlay listing all shortcuts (you already have it; just add a one-keystroke reveal).
  • Fullscreen-only shortcuts — keep the existing overlay; ensure it shows only in fullscreen and dismisses on Escape.
  • AirPlay / Cast buttons — detect support, show icons only when available.
08
Reliability & cleanup
Fix the object-URL leak. Add error handling. Refactor if it hurts.
Critical bug
+
  • Object URL revocation — URL.revokeObjectURL is never called. Add a URL.revoke(item.url) in the playlist-remove path and on library clear. Currently a memory leak.
  • Media error handling — show a friendly error in the action popup when video.error is non-null; auto-skip to next on fatal decode errors.
  • Drag-drop visual feedback — highlight the page while a file is being dragged over it.
  • Tab visibility behavior — option to pause when tab is hidden (default off; users get annoyed when music stops).
  • Refactor — break player-mute.html into logical sections via comments / <script type="module"> extraction. Optional — only if the file is getting hard to navigate.
Recommended cut If you only do three.

The minimum set that gets you the largest quality-of-life jump without committing to the whole plan.

  1. Phase 1 — persistence
    Biggest quality-of-life win. The single feature that makes the player feel "real" instead of a scratchpad.
  2. Phase 8 — object URL leak fix
    It's a real bug; cleanup is 10 lines. The only item in this list that's a defect, not a feature.
  3. Phase 2's volume keys + Phase 4's subtitle toggle
    They round out the shortcuts and the existing media properties popup. ↑/↓ for volume, C for captions.
